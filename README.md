Smart Toll Gate System using STM32
Project Description

This project demonstrates an automated toll gate control system built using the STM32 Nucleo development board, an ultrasonic sensor, and a servo motor. The system is designed to simulate a real-world toll gate mechanism where the entry and exit of vehicles are handled automatically without human intervention. The ultrasonic sensor continuously monitors the distance of objects in front of it, and when a vehicle is detected within a predefined range, the STM32 microcontroller processes this input and generates a PWM signal to control a servo motor. The servo motor acts as the gate barrier, opening when a vehicle is detected and closing after a short delay. This project highlights the integration of sensing, processing, and actuation in embedded systems.

Objectives

The main objectives of this project are to interface an ultrasonic sensor with the STM32 microcontroller for distance measurement, generate PWM signals to control a servo motor, implement decision-making logic based on distance thresholds, and automate the opening and closing of a gate mechanism. Additionally, the project aims to demonstrate real-time embedded control and hardware-software interaction.

Hardware Used

The hardware components used in this project include the STM32 Nucleo Development Board, an HC-SR04 Ultrasonic Sensor, a Servo Motor (SG90 or equivalent), jumper wires, a breadboard, and a power supply. The STM32 board acts as the central controller, the ultrasonic sensor performs object detection, and the servo motor provides the mechanical gate movement.

Pin Configuration

The ultrasonic sensor’s TRIG pin is connected to PA1 configured as a GPIO output, and the ECHO pin is connected to PA2 configured as a GPIO input. The servo motor signal pin is connected to PA8, which is configured as a PWM output using a timer channel (TIM1_CH1). The power and ground connections are provided appropriately to all components.

System Functionality

The system operates by continuously measuring the distance between the ultrasonic sensor and any object placed in front of it. The STM32 generates a trigger pulse to the ultrasonic sensor, which emits ultrasonic waves and waits for the echo signal to return. If the measured distance is less than a predefined threshold (for example, 10 cm), the system assumes a vehicle is present. In response, the STM32 generates a PWM signal to rotate the servo motor to approximately 90 degrees, thereby opening the gate. After a fixed delay to allow the vehicle to pass, the servo is rotated back to 0 degrees, closing the gate. This process repeats continuously, enabling automatic gate operation.

Output Behavior

The output of the system is observed through the motion of the servo motor, which acts as the toll gate barrier. When a vehicle is detected, the gate opens smoothly, and after a delay, it closes automatically. This provides a clear and visible demonstration of automated control based on sensor input.

Software Details

The project is implemented in the C programming language using STM32CubeIDE. STM32 HAL (Hardware Abstraction Layer) drivers are used for configuring and controlling peripherals such as GPIO, timers, and PWM generation. The ultrasonic sensor interfacing logic is implemented using GPIO control and precise timing functions, while the servo motor is controlled using PWM signals generated by the timer peripheral.

How to Run

To run the project, open the project in STM32CubeIDE and configure the peripherals as specified in the pin configuration. Connect the ultrasonic sensor and servo motor to the STM32 board according to the circuit diagram. Build and flash the code onto the STM32 Nucleo board using a USB connection. Once powered, the system will start measuring distance continuously. When an object is placed within the threshold distance, the servo motor will rotate to open the gate and automatically close it after a short delay.
